# 4.6 Explicit Chain-of-Thought (CoT)

## Definition & Architectural Decision

Explicit CoT inserts a reasoning step before each tool call: the agent first generates free-text thinking, then uses that context to inform the subsequent tool call. This is available in all benchmarks or documented as calendar-only.

---

## Per-Package Analysis

### Calendar

**Location:** `calendar_scheduling/cli.py` lines 320-343

Calendar has a global flag and two per-agent overrides:

| Flag | Default | Description |
|------|---------|-------------|
| `--explicit-cot` | None (required) | Enable/disable CoT for all agents (`true`/`false`) |
| `--assistant-explicit-cot` | None | Override for assistant agent |
| `--requestor-explicit-cot` | None | Override for requestor agent |

The `--explicit-cot` flag is **required** for new runs (validation at line 445).

**Implementation:** In `agents/calendar_base.py`, the `CalendarAgent.__init__()` accepts `explicit_cot: bool = False` (line 46). When enabled, `generate_tool_call()` (line 173) calls `_generate_cot_reasoning()` (lines 153-171) before each tool-call generation:

1. A user message is appended: "Before taking your next action, think carefully about what should be the one next action..." (line 162)
2. The model generates free-text reasoning (no tools)
3. The reasoning is injected as an assistant message into the context
4. Then the actual tool-call generation happens with the thinking in context
5. On success, the CoT message is appended to the permanent message history (lines 234-237)

CoT reasoning is regenerated on each retry attempt within `generate_tool_call()` (line 181: "Generate CoT reasoning for every retry if enabled").

Both `assistant_explicit_cot` and `requestor_explicit_cot` are stored in `RunConfig` (lines 70-77) and `BenchmarkMetadata` (lines 447-458) for reproducibility.

### Form-Filling

**Confirmed: No explicit CoT support.** Grepping for `explicit_cot` or `explicit-cot` across the entire `form_filling/` package returns zero matches.

Form-filling does have a conceptually related mechanism: the `SendMessageParams.thinking` field (in `agents/assistant.py` line 25-27) forces the agent to produce internal reasoning via structured output. However, this is fundamentally different:
- It is a tool parameter (structured output), not a separate LLM call
- It happens within the tool call, not before it
- It is always present (not toggleable)
- The `thinking` field content is hidden from the interviewer

The `SendMessageSocialReasoningParams` variant (lines 33-69) extends this with `task_related_thinking`, `tom_thinking_interviewer`, and `tom_thinking_principal` fields.

### Marketplace

**Confirmed: No explicit CoT support.** Grepping for `explicit_cot` or `explicit-cot` across the entire `marketplace/` package returns zero matches.

The `MarketplaceAgent.generate_tool_call()` in `marketplace_base.py` (line 105) directly generates tool calls without any preceding reasoning step. There is no mechanism to inject a thinking phase before tool selection.

**Summary:** Explicit CoT is currently calendar-only. Form-filling achieves a similar effect through structured `thinking` fields in tool parameters but with different mechanics. Marketplace has no reasoning augmentation.
