# 4.4 Per-Agent Reasoning Effort

## Definition & Architectural Decision

All benchmarks support per-agent reasoning effort flags, allowing fine-grained control over how much "thinking" each agent role uses. Values can be named levels (`none`, `minimal`, `low`, `medium`, `high`, `xhigh`, `default`) or integer budget tokens. All packages use the shared `parse_reasoning_effort` utility from `shared/cli_utils.py`.

---

## Per-Package Analysis

### Calendar

**Location:** `calendar_scheduling/cli.py` lines 293-318

Calendar has the most granular reasoning effort controls, with a global default and three per-agent overrides:

| Flag | Default | Description |
|------|---------|-------------|
| `--reasoning-effort` / `-r` | None | Default reasoning effort for all agents |
| `--assistant-reasoning-effort` | None | Override for assistant agent |
| `--requestor-reasoning-effort` | None | Override for requestor agent |
| `--judge-reasoning-effort` | None | Override for judge |

All four use `parse_reasoning_effort` (imported from `shared/cli_utils.py` at line 12) as the `type` argument. The per-agent values override the global `--reasoning-effort` when both are specified. These are stored in `RunConfig` (`checkpoints/run_config.py` lines 47-56) and serialized into `BenchmarkMetadata` (`types.py` lines 451-462) for reproducibility.

The values flow into `ModelClient` construction, which passes them through to the underlying LLM API.

### Form-Filling

**Location:** `form_filling/cli.py` lines 98-115

Form-filling has three per-agent flags but no global default:

| Flag | Default | Description |
|------|---------|-------------|
| `--interviewer-reasoning-effort` | None | Reasoning effort for interviewer agent (interactive mode) |
| `--assistant-reasoning-effort` | None | Reasoning effort for assistant agent or form filler |
| `--judge-reasoning-effort` | None | Reasoning effort for judge |

All three use `parse_reasoning_effort` from `shared/cli_utils.py` (imported at line 9).

**Differences from Calendar:**
- No global `--reasoning-effort` fallback. Each agent must be configured individually.
- The "requestor" role is called "interviewer" in form-filling, reflecting different domain terminology.
- Values are passed directly to `run_tasks()` (`runner.py` lines 119-121) rather than stored in a `RunConfig` model.

### Marketplace

**Location:** `marketplace/cli.py` lines 53-57

Marketplace has a single global flag:

| Flag | Default | Description |
|------|---------|-------------|
| `--reasoning-effort` | None | Reasoning effort for both buyer and seller models |

**Key difference:** This flag does **not** use `parse_reasoning_effort` from shared utilities. Instead, it applies raw string-or-int parsing inline (`cli.py` lines 86-91):
```python
try:
    reasoning_effort = int(args.reasoning_effort)
except ValueError:
    reasoning_effort = args.reasoning_effort
```

There are no per-agent overrides (`--buyer-reasoning-effort`, `--seller-reasoning-effort`) despite having per-agent model flags (`--buyer-model`, `--seller-model`). Both the buyer and seller `ModelClient` instances receive the same `reasoning_effort` value (lines 93-100).

**Gaps:**
- No per-role reasoning effort (buyer vs seller).
- No judge reasoning effort (marketplace has no LLM judge).
- Does not use the shared `parse_reasoning_effort` utility, so named levels like `xhigh` may not be recognized consistently.
