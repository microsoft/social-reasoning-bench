# 1.5 Remove AskUser

## Definition & Architectural Decision

**Decision:** Form-filling agents cannot ask the user for information. File search (SearchEmail, ReadEmail, SearchCalendar, ReadCalendar) is the only tool for gathering info.

The `AskUser` tool currently allows the form-filling agent to ask the user (simulated by `OracleUser`) for field values it cannot find in the artifacts. This creates a shortcut that bypasses the due diligence challenge: instead of searching through emails and calendars to find information, the agent can simply ask. Removing `AskUser` makes the benchmark test whether agents can independently gather information from structured data sources.

## Per-Package Analysis

### Form-Filling -- Current AskUser Integration

#### One-Shot Mode

**File:** `packages/sage-benchmark/sage_benchmark/form_filling/one_shot.py`

- `AskUser` is defined as a tool at lines 212-232 of `one_shot.py`:
  ```python
  {
      "type": "function",
      "function": {
          "name": "ask_user",
          "description": "Ask the user about fields in the form. ...",
          "parameters": {
              "type": "object",
              "properties": {
                  "question": {"type": "string", "description": "..."}
              },
              "required": ["question"],
          },
      },
  },
  ```
- The `ASK_USER_THINKING_SUFFIX` (lines 89-91) is appended to the thinking prompt: "If you don't have enough information in the artifacts for a particular field, use the ask_user tool to ask the user about it."
- The ask-user loop (lines 277-310): When the agent calls `ask_user`, the question is forwarded to `oracle_user.answer_question()` (line 281), the answer is appended to the conversation, and the loop continues for up to `max_ask_rounds` (default 50).
- The `user_qa_history` list (line 244) records all Q&A exchanges and is stored in `TaskExecutionResult.user_qa_history` (line 382).

#### Interactive Mode

**File:** `packages/sage-benchmark/sage_benchmark/form_filling/interactive.py`

- The assistant agent has `AskUser` in its tool list (see `assistant.py`).
- In `_run_conversation_loop()` (lines 121-212), the AskUser handling is at lines 175-192:
  ```python
  if tool_name == "AskUser":
      question = arguments.get("question", "")
      ask_user_count += 1
      if ask_user_count > max_ask_rounds:
          assistant.add_tool_result("You have reached the maximum...")
          break
      user_answer = await oracle_user.answer_question(question)
      user_qa_history.append({"question": question, "answer": user_answer})
      assistant.add_tool_result(user_answer)
      continue  # Let assistant generate another action
  ```
- AskUser is a "side channel" -- the response goes only to the assistant, not to the interviewer. This is distinct from `SendMessage` (which goes to the interviewer) and `EndConversation`.

#### OracleUser

**File:** `packages/sage-benchmark/sage_benchmark/form_filling/agents/oracle_user.py`

- `OracleUser` (line 17) is the simulated user that answers AskUser questions.
- Constructor (lines 22-51): Takes `unmasked_ground_truth` dict, separates close-ended fields (with known answers) from open-ended fields (which defer to artifacts).
- `answer_question()` (line 56):
  1. Uses LLM to parse which form field(s) the question refers to via `build_question_field_mapping_prompt()` (line 66).
  2. Performs deterministic lookup: if the mapped field is close-ended and has a known answer, returns the value; if open-ended, returns "Check the artifacts available and answer"; if unknown, returns "I don't know."
- The oracle is a testing affordance: it provides ground-truth answers to simulate what a real user would know.

#### Due Diligence Evaluation

**File:** `packages/sage-benchmark/sage_benchmark/form_filling/evaluation/due_diligence/evaluate.py`

- `evaluate_due_diligence()` (line 71) directly uses `user_qa_history` to compute ask-user metrics:
  - **Ask-user P/R/F1** (lines 201-214): TP = asked unfindable field, FP = asked findable field, FN = unfindable not asked.
  - **Masked field accuracy** (lines 197-199): Fraction of masked fields answered correctly.
- These metrics assume AskUser exists. Removing AskUser means:
  - `user_qa_history` will always be empty.
  - Ask-user P/R/F1 will be undefined (0/0 divisions).
  - Masked field accuracy becomes the sole due diligence metric.
  - The `question_field_mappings` logic (lines 119-153) becomes unused.

#### Schemas

**File:** `packages/sage-benchmark/sage_benchmark/form_filling/schemas.py`

- `TaskExecutionResult.user_qa_history` (line 263): `list[dict[str, str]]` storing Q&A exchanges.
- `InteractiveTaskExecutionResult.user_qa_history` (line 475): same field for interactive mode.
- `DueDiligenceEvaluation` (line 337): Contains `ask_user_precision`, `ask_user_recall`, `ask_user_f1`, `total_ask_user_calls`, and `question_field_mappings` -- all tied to AskUser.

### Calendar

Calendar does **not** have AskUser. The assistant agent uses only calendar-specific tools: `SendEmail`, `GetEmails`, `ListMeetings`, `ListContacts`, `RequestMeeting`, `ReplyMeeting`, `Wait`, and `EndConversation` (defined in `calendar_scheduling/environment/actions.py`). There is no user-query side channel.

### Marketplace

Marketplace does **not** have AskUser. Agents have: `SendMessage`, `GetMessages`, `MakeOffer`, `AcceptOffer`, `Wait`, and `EndNegotiation` (defined in `marketplace/environment/actions.py`). There is no user-query side channel.

## Impact of Removal

Removing AskUser requires changes in the following areas:

1. **`one_shot.py`:** Remove the `ask_user` tool definition (lines 212-232), the `ASK_USER_THINKING_SUFFIX` (lines 89-91), and the ask-user handling loop (lines 277-310). The `oracle_user` parameter becomes unused.
2. **`interactive.py`:** Remove AskUser handling from `_run_conversation_loop()` (lines 175-192). The `oracle_user` parameter and `max_ask_rounds` become unused.
3. **`agents/assistant.py`:** Remove AskUser from `_build_tools()` (lines 101-123). Update `generate_action()` docstring.
4. **`agents/oracle_user.py`:** The entire module becomes unused and can be removed.
5. **`evaluation/due_diligence/evaluate.py`:** Remove ask-user P/R/F1 metrics. Redefine due diligence around file-search tool calls (SearchEmail, ReadEmail, SearchCalendar, ReadCalendar) as the "unit of work."
6. **`schemas.py`:** `user_qa_history` field becomes vestigial. Ask-user fields on `DueDiligenceEvaluation` should be removed or deprecated.
