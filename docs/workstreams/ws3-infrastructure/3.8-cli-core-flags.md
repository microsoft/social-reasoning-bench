# 3.8 Consistent Core CLI Flags

## Definition & Architectural Decision

All benchmarks must expose a consistent set of core CLI flags with uniform names, semantics, and defaults:

| Flag | Type | Default | Description |
|---|---|---|---|
| `--data` | `str` (nargs="+") | required | Data files or directories |
| `--model` | `str` | required | Default model for all agents |
| `--judge-model` | `str` | None | Model for evaluation (overrides --model) |
| `--batch-size` | `int` | 32 | Parallel task count |
| `--limit` | `int` | None | Limit number of tasks |
| `--output-dir` | `Path` | benchmark-specific | Output directory |
| `--log-style` | `str` | "verbose" | Logger style: verbose/progress/quiet |
| `--reasoning-effort` | `str\|int` | None | Default reasoning effort |
| Per-agent model overrides | `str` | None | `--<role>-model` overrides `--model` |

### Form-Filling

**File:** `packages/sage-benchmark/sage_benchmark/form_filling/cli.py` (`parse_args()`, line 12)

| Flag | Present | Name | Default | Notes |
|---|---|---|---|---|
| `--data` | Yes | `--data` | required | Single path, not `nargs="+"` |
| `--model` | No | -- | -- | **Missing; uses `--assistant-model` only** |
| `--judge-model` | Yes | `--judge-model` | "gpt-4.1" | **Hard-coded default** |
| `--batch-size` | Yes | `--batch-size` | 75 | **Different default** (75 vs 32) |
| `--limit` | Yes | `--limit` | None | |
| `--output-dir` | Yes | `--output-dir` | "outputs/form_filling" | |
| `--log-style` | No | -- | -- | **Missing** |
| `--reasoning-effort` | Partial | `--assistant-reasoning-effort`, `--interviewer-reasoning-effort`, `--judge-reasoning-effort` | None | **No global `--reasoning-effort`** |
| Agent overrides | Partial | `--assistant-model`, `--interviewer-model` | None | No `--model` default |

Additional form-filling-specific flags:
- `--execution-mode` (one-shot/interactive/gui)
- `--run-mode` (all/tasks/eval), `--task-results-path`
- `--prompt-type`, `--max-rounds`, `--interviewer-type`, `--single-field-mode`
- `--malicious-strategy`, `--malicious-attack-type`, `--malicious-strategies-file`
- `--assistant-temperature`, `--file-system`, `--social-reasoning`, `--use-privacy-example`
- `--http-port`, `--max-gui-steps`, `--restructure-model`
- `--base-url`, `--max-concurrent-requests`, `--id`
- `--interviewer-form-fill-model`

**Gaps:**
1. No `--model` flag; must add as default for `--assistant-model` and `--interviewer-model`.
2. No `--log-style` flag; needs `BenchmarkLogger` integration (see 3.4).
3. `--judge-model` has a hard-coded default ("gpt-4.1") instead of falling back to `--model`.
4. `--data` takes a single string, not `nargs="+"`.
5. `--batch-size` default is 75, should align to 32 or be documented as intentionally different.
6. No global `--reasoning-effort`; only per-agent variants exist.

6. No `--run-mode` support (see 3.6).
