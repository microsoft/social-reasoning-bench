# 3.5 Structured Output Models for Marketplace

## Definition & Architectural Decision

Marketplace must define `BenchmarkOutput` and `BenchmarkSummary` Pydantic models that aggregate all metrics into a typed, serializable structure. This replaces the current raw `dict` summary and ensures marketplace output is programmatically consumable, consistent with the other benchmarks, and extensible for new metrics.

---

## Per-Package Analysis

### Calendar (Reference Implementation)

**File:** `packages/sage-benchmark/sage_benchmark/calendar_scheduling/types.py`

Calendar defines two structured output models:

1. **`BenchmarkSummary`** (line 487): 20+ typed fields covering:
   - Task counts: `total_tasks`, `valid_tasks`, `failed_tasks`, `eval_error_tasks`, `tasks_hit_max_rounds`
   - Task completion: `task_successes`, `task_success_rate`
   - Failure breakdown: `task_conflicts`, `task_illegal_moves`, `task_missing_meetings`, `task_unwanted_meetings`
   - Privacy: `privacy_avg_leakage_rate`, `privacy_tasks_with_leakage`, `privacy_leakage_rate`
   - Fiduciary: `fiduciary_tasks_with_preferences`, `fiduciary_avg_preference_score`, `fiduciary_avg_assistant_duty_of_care_score`, etc.
   - Due diligence: `due_diligence_avg_message_count`, `due_diligence_avg_preference_mention_count`, `due_diligence_avg_proposal_count`
   - Error details: `failed_task_errors: list[FailedTaskError]`, `eval_errors: list[FailedTaskError]`

2. **`BenchmarkOutput`** (line 523): Top-level output containing:
   - `metadata: BenchmarkMetadata` -- run configuration and timing
   - `summary: BenchmarkSummary` -- aggregate statistics
   - `results: list[TaskEvaluationResult]` -- per-task results

`BenchmarkMetadata` (line 428) captures model names, batch size, task count, system prompt, reasoning effort settings, and elapsed time.

### Form-Filling

**File:** `packages/sage-benchmark/sage_benchmark/form_filling/runner.py`

Form-filling writes three separate JSON files rather than a single structured output:
- `task_results.json` (line 189): Raw execution results.
- `eval_results.json` (line 190): Evaluation results.
- `summary.json` (line 191): Summary dict built inline.

The summary is constructed as a raw dict with inline computation (lines 1086-1145 for one-shot, lines 1317-1389 for interactive). The dict includes correctness metrics (precision/recall/F1), privacy scores, CI violations, due diligence, and duty of care -- but these are not captured in a Pydantic model.

**Status:** Partially structured. Has `FormTaskEvaluationResult` and `InteractiveTaskEvaluationResult` Pydantic models for per-task results, but no `BenchmarkOutput` or `BenchmarkSummary` model. Summary is an untyped dict.

### Marketplace (Current State)

**File:** `packages/sage-benchmark/sage_benchmark/marketplace/cli.py` (lines 61-67)

The entire summary is a raw dict with only 3 fields:

```python
def _compute_summary(results: list[TaskExecutionResult]) -> dict:
    deal_count = sum(1 for r in results if r.outcome.deal_reached)
    return {
        "task_count": len(results),
        "deals_reached": deal_count,
        "deal_rate": (deal_count / len(results)) if results else 0.0,
    }
```

The output file (`results.json`, lines 127-138) is a raw dict with `task_count`, `summary`, and `results` keys. There is no `BenchmarkMetadata` capturing model names, timing, or configuration.

**Gaps to address:**

1. **Define `BenchmarkSummary`** Pydantic model with fields for:
   - `task_count`, `deals_reached`, `deal_rate` (existing)
   - `avg_deal_price`, `avg_rounds`, `tasks_ended_by_buyer`, `tasks_ended_by_seller`, `tasks_hit_max_rounds`
   - Privacy metrics: `buyer_avg_estimation_error`, `seller_avg_estimation_error`
   - Invalid action counts
   - Error tracking: `failed_tasks`, `failed_task_errors`

2. **Define `BenchmarkMetadata`** with:
   - `buyer_model`, `seller_model`, `timestamp`, `max_rounds`, `max_steps_per_turn`, `task_count`
   - `reasoning_effort`, `elapsed_seconds`

3. **Define `BenchmarkOutput`** combining `metadata`, `summary`, and per-task results.

4. **Use `compute_evaluation_summary()`** function analogous to calendar's `evaluator.py` `compute_evaluation_summary()` (line 395).
