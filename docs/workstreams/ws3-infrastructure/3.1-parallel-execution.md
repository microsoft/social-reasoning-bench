# 3.1 Parallel Execution via TaskPoolExecutor

## Definition & Architectural Decision

All benchmarks must use `TaskPoolExecutor` from `shared/executors.py` with cooperative cancellation and a `--batch-size` CLI flag. `TaskPoolExecutor` provides continuous batching: it maintains approximately `batch_size` concurrent tasks at all times by starting new tasks as soon as others complete, rather than processing in fixed-size batches with idle time between them.

---

## Per-Package Analysis

### Shared (Reference Implementation)

**File:** `packages/sage-benchmark/sage_benchmark/shared/executors.py`

`TaskPoolExecutor` (line 18) provides:

- **Constructor parameters** (lines 36-44): `batch_size`, `on_task_complete` callback, `on_task_error` callback, `task_logger`, `quiet_cancel`, and `cancel_event` for cooperative cancellation.
- **`run()` method** (line 63): Accepts an iterable of coroutines (can be a generator), maintains a pool of `batch_size` concurrent `asyncio.Task` objects, and collects results in completion order.
- **Cooperative cancellation** (lines 83-109): When the `cancel_event` is set, the executor force-cancels all pending tasks and waits up to 5 seconds for cleanup.
- **`asyncio.CancelledError` handling** (lines 169-188): If the executor itself is cancelled, it cleans up pending tasks with a timeout.

### Form-Filling

**File:** `packages/sage-benchmark/sage_benchmark/form_filling/runner.py`

Form-filling uses `asyncio.gather()` in fixed-size batches with manual slicing (lines 349-379):

```python
for batch_start in range(0, len(tasks), batch_size):
    batch = tasks[batch_start:batch_end]
    batch_results = await asyncio.gather(*batch_tasks, return_exceptions=True)
```

This pattern appears repeatedly for one-shot mode (lines 349-379), GUI mode, and interactive mode. The approach has several drawbacks versus `TaskPoolExecutor`:

1. **Idle time between batches**: All tasks in a batch must complete before the next batch starts.
2. **No cooperative cancellation**: Ctrl+C kills tasks abruptly.
3. **No continuous refilling**: If one task finishes early, its slot sits idle until the entire batch completes.

Form-filling exposes `--batch-size` in the CLI (`cli.py`, line 70, default: 75).

**Gap:** Replace all `asyncio.gather()` batch loops with `TaskPoolExecutor.run()`. The `on_task_complete` callback can replace the manual result-accumulation logic.

### Marketplace

**File:** `packages/sage-benchmark/sage_benchmark/marketplace/runner.py`

The `run_tasks()` function (line 157) executes tasks **sequentially** in a plain `for` loop:

```python
async def run_tasks(...) -> list[TaskExecutionResult]:
    results: list[TaskExecutionResult] = []
    for task in tasks:
        result = await _run_single_task_llm(task, ...)
        results.append(result)
```

There is no `--batch-size` CLI flag in `marketplace/cli.py`.

**Gap:** Marketplace has no parallelism at all. It needs:
1. Adoption of `TaskPoolExecutor` in `run_tasks()`.
2. A `--batch-size` CLI argument in `marketplace/cli.py`.
3. A `cancel_event` parameter for cooperative cancellation.
