# 3.7 Per-Task Results and Evaluation Summary Printing

## Definition & Architectural Decision

All benchmarks must print formatted per-task results and evaluation summaries using dedicated functions analogous to calendar's `print_per_task_summary()` and `print_evaluation_summary()`. This ensures consistent, scannable output across all benchmarks.

---

## Per-Package Analysis

### Calendar (Reference Implementation)

**File:** `packages/sage-benchmark/sage_benchmark/calendar_scheduling/evaluation/evaluator.py`

Calendar provides two well-structured printing functions:

1. **`print_per_task_summary()`** (line 172): Prints a formatted table with column groups:
   - **Task description** (45 chars): task ID, malicious status, expected outcome, validity marker.
   - **Task Success** group: Scheduled, Conflicts, Illegal, Success columns.
   - **Privacy** group: Leakage rate column.
   - **Fiduciary** group: Preference score, Assistant Duty of Care, Duration Error columns.
   - **Due Diligence** group: Message count, Preference mentions, Proposal count columns.

   The table has aligned column headers, group headers with dash separators, and rows sorted by task ID.

2. **`print_evaluation_summary()`** (line 310): Prints aggregate statistics:
   - Task counts (total, valid, failed, eval errors, max rounds hit)
   - Success rate
   - Task failure breakdown (conflicts, illegal moves, missing/unwanted meetings)
   - Privacy breakdown (avg leakage rate, tasks with leakage)
   - Fiduciary breakdown (preference scores, duty of care)
   - Due diligence breakdown (message counts, preference mentions, proposals)

Both are called from `cli.py` (lines 601, 695-696).

### Form-Filling

**File:** `packages/sage-benchmark/sage_benchmark/form_filling/runner.py`

Form-filling prints summaries inline within the `_run_one_shot_mode()` and `_run_interactive_mode()` functions using long blocks of `print()` statements (lines 1086-1145 for one-shot, lines 1317-1389 for interactive).

For one-shot mode, the summary covers:
- Execution stats (total tasks, successful executions)
- Correctness metrics (precision, recall, F1, perfect forms)
- Privacy metrics (average privacy score, leakage rate, secret counts)
- CI violations
- Due diligence metrics (masked field accuracy, ask-user precision/recall/F1)
- Duty of care metrics
- Validation rate

There is no dedicated `print_per_task_summary()` function. Per-task results are printed inline during batch processing (`print(f"Task {result.task_index}: {result.action.action_type.upper()} - Form {result.task.form_id}")` at line 416).

The visualization module (`visualization_result.py`) generates HTML reports for individual tasks, but this is separate from CLI output.

**Gap:** Extract the inline print blocks into dedicated `print_per_task_summary()` and `print_evaluation_summary()` functions. Create a tabular per-task view showing form ID, action type (fill/reject), correctness F1, privacy score, and validation status.

### Marketplace

**File:** `packages/sage-benchmark/sage_benchmark/marketplace/cli.py` (lines 140-143)

Marketplace summary output is minimal:

```python
print("Marketplace experiment finished.")
print(f"Loaded {len(tasks)} task(s)")
print(f"Saved results to {out_path}")
print(f"Summary: deal_rate={summary['deal_rate']:.1%}")
```

Per-task output is a single callback line (lines 102-106):

```python
print(
    f"Finished task {r.task.id}: deal={r.outcome.deal_reached} "
    f"price={r.outcome.deal_price} invalid_actions={r.invalid_actions}"
)
```

There is no tabular per-task view and no structured evaluation summary.

**Gap:** Implement `print_per_task_summary()` with columns for task ID, deal reached, deal price, buyer/seller surplus, invalid actions, and privacy probe accuracy. Implement `print_evaluation_summary()` aggregating deal rate, average prices, surplus distribution, and privacy metrics.
