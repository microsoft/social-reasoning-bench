# 2.3 Remove Calendar Artifact Injection System

## Definition & Architectural Decision

Calendar's artifact injection system (`EmailThread`, `Note`, `EventReference`) is unused infrastructure that should be removed. These types were designed to inject additional context (e.g., prior email threads, personal notes) into the assistant's conversation to hint at which meetings are movable or secret, but the feature is entirely opt-in via a `--artifacts` CLI flag that points to a separate JSON file. No such JSON files ship with the benchmark data, and no experiment configurations reference artifacts. Removing this dead code simplifies the codebase and reduces the surface area for confusion.

---

## Per-Package Analysis

### Calendar

#### Type definitions

**File:** `packages/sage-benchmark/sage_benchmark/calendar_scheduling/types.py` (lines 244-286)

Four types support artifacts:

- `EventReference` (line 247): Links an artifact to calendar events, with `hints_movable` and `hints_secret` booleans.
- `EmailMessage` (line 255): A single message in an email thread with `sender` and `content`.
- `EmailThread` (line 262): Discriminated on `artifact_type: Literal["email"]`. Contains `date`, `subject`, `messages: list[EmailMessage]`, and `event_references: list[EventReference]`.
- `Note` (line 274): Discriminated on `artifact_type: Literal["note"]`. Contains `date`, `title`, `content`, and `event_references`.
- `Artifact` (line 286): A discriminated union `Annotated[Union[EmailThread, Note], Discriminator("artifact_type")]`.

#### Agent injection

**File:** `packages/sage-benchmark/sage_benchmark/calendar_scheduling/agents/assistant/calendar_assistant.py` (lines 38-60, 72, 90-92)

- `format_artifacts_for_context()` (line 38) renders a list of `Artifact` objects into a natural-language context message with sections for email threads and notes.
- `CalendarAssistantAgent.__init__()` accepts `artifacts: list[Artifact] | None` (line 72) and injects the formatted string as a user message before the instruction (lines 90-92).

#### Loader

**File:** `packages/sage-benchmark/sage_benchmark/calendar_scheduling/loader.py` (lines 93-112)

- `load_artifacts()` reads a JSON file, validates each entry via `TypeAdapter(Artifact)`, and returns `dict[int, list[Artifact]]` mapping task IDs to artifact lists.

#### CLI flag

**File:** `packages/sage-benchmark/sage_benchmark/calendar_scheduling/cli.py` (lines 249-253)

- `--artifacts` argument: `parser.add_argument("--artifacts", default=None, help="Path to artifacts JSON file...")`.

#### RunConfig

**File:** `packages/sage-benchmark/sage_benchmark/calendar_scheduling/checkpoints/run_config.py` (line 81)

- `artifacts: str | None = Field(default=None, ...)` stored in `RunConfig` and passed through `from_args()` (line 140).

#### Runner

**File:** `packages/sage-benchmark/sage_benchmark/calendar_scheduling/runner.py` (line 216, 287)

- `run_single_task()` accepts `artifacts: list[Artifact] | None` (line 216) and passes it to `CalendarAssistantAgent` (line 287).

#### Experiment infrastructure

**Files:**
- `packages/sage-benchmark/sage_benchmark/calendar_scheduling/experiments/setup.py` (lines 38-57): `load_experiment_tasks()` accepts `artifacts_path` and calls `load_artifacts()`.
- `packages/sage-benchmark/sage_benchmark/calendar_scheduling/experiments/runner.py` (lines 116-119, 130, 250, 516): The `Experiment` class stores `self.artifacts_by_task` and passes per-task artifacts into `run_single_task()`.

#### Summary of removal scope

All of the above are artifact-related code paths that can be removed:

| Location | What to remove |
|---|---|
| `types.py` lines 244-286 | `EventReference`, `EmailMessage`, `EmailThread`, `Note`, `Artifact` |
| `calendar_assistant.py` lines 38-60, constructor param | `format_artifacts_for_context()`, `artifacts` parameter |
| `loader.py` lines 93-112 | `load_artifacts()` function |
| `cli.py` lines 249-253 | `--artifacts` argument |
| `run_config.py` line 81 | `artifacts` field |
| `runner.py` line 216, 287 | `artifacts` parameter threading |
| `experiments/setup.py` lines 38-57 | `artifacts_path` parameter, `load_artifacts` call |
| `experiments/runner.py` lines 116-130, 250, 516 | `artifacts_by_task` storage and usage |

### Form-Filling

Not applicable. Form-filling has its own artifact system (`ArtifactData` in `schemas.py`) that is actively used to provide persona documents to the agent. These are a core part of the form-filling benchmark and should be kept.

### Marketplace

Not applicable. Marketplace has no artifact system.
