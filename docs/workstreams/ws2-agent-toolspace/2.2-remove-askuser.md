# 2.2 Remove AskUser Tool

## Definition & Architectural Decision

The `AskUser` tool should not exist in the agent toolspace. From a toolspace perspective, AskUser creates ambiguity: it lets the agent defer decisions to the principal mid-task, which undermines the benchmark's ability to measure autonomous agent behavior. This document covers the toolspace registration and definition angle; see **1.5** for the full analysis of evaluation impact.

---

## Per-Package Analysis

### Calendar

**File:** `packages/sage-benchmark/sage_benchmark/calendar_scheduling/environment/actions.py`

No `AskUser` tool exists. The calendar benchmark has no mechanism for the agent to query its principal during a task. The assistant receives its instruction once and acts autonomously.

**Status:** Already compliant -- no action needed.

---

### Form-Filling

**Files:**
- `packages/sage-benchmark/sage_benchmark/form_filling/agents/assistant.py` (lines 103-123)
- `packages/sage-benchmark/sage_benchmark/form_filling/one_shot.py` (lines 211-232)
- `packages/sage-benchmark/sage_benchmark/form_filling/prompts.py` (line 193)

`AskUser` is registered in two places:

1. **Interactive mode** (`assistant.py`, line 104): Defined as a raw OpenAI function-tool dict within the `_build_tools()` function. It has a single `question: str` parameter and is described as "Ask the user about fields in the form." It sits alongside `SendMessage` and `EndConversation` in the tool list returned at line 84-124.

2. **One-shot mode** (`one_shot.py`, line 211): Defined as `ask_user` (lowercase) with identical semantics. Registered in the tools list alongside `fill_form` and `reject_form`.

3. **Prompt references** (`prompts.py`, line 193): The interactive instruction text explicitly lists `AskUser` as an available tool.

The tool is handled at runtime in `interactive.py` (line 175) where `AskUser` calls are routed to an `OracleUser` that answers from ground-truth data.

**Gap:** The `AskUser` / `ask_user` tool must be removed from both `_build_tools()` in `assistant.py` and the tools list in `one_shot.py`. Prompt text referencing it (`prompts.py` line 193, line 413, line 430) must also be updated. The `OracleUser` infrastructure can remain for evaluation purposes but should not be exposed as an agent-callable tool. See 1.5 for the evaluation-side implications.
