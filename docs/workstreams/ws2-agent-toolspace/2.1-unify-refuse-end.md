# 2.1 Unify Refusal / Termination Tool Name

## Definition & Architectural Decision

All benchmarks must use a single, consistently named tool -- `EndConversation(reason)` -- as the refusal and termination mechanism. The tool accepts a `reason: str` parameter and terminates the agent loop. This gives every benchmark a uniform signal for "the agent chose to stop" regardless of whether it is refusing a request, ending a completed interaction, or walking away from a negotiation.

---

## Per-Package Analysis

### Calendar

**File:** `packages/sage-benchmark/sage_benchmark/calendar_scheduling/environment/actions.py`

Already compliant. The `EndConversation` class (line 194) inherits from `Tool` and carries a single field:

```python
class EndConversation(Tool):
    reason: str = Field(description="Reason for ending the conversation")
```

`EndConversation` is **not** part of the `CalendarAction` union (line 216-226) because it is appended separately in the assistant agent constructor (`calendar_assistant.py`, line 81: `tools=CALENDAR_TOOLS + [EndConversation]`). This keeps it available to the assistant but not to the requestor.

**Status:** Matches the target design.

---

### Form-Filling

**Files:**
- `packages/sage-benchmark/sage_benchmark/form_filling/one_shot.py` (lines 41-44, 193-209)
- `packages/sage-benchmark/sage_benchmark/form_filling/agents/assistant.py` (lines 72-76, 84-99)

Two separate termination mechanisms exist:

1. **One-shot mode** uses `reject_form` (line 205). This is a raw OpenAI function-tool dict with `RejectFormParams(reason: str)` (line 41-44). It serves the same purpose as `EndConversation` -- the agent declines to proceed -- but uses a different name.

2. **Interactive mode** uses `EndConversation` (line 95-99 in `assistant.py`). The tool is defined via `EndConversationParams(reason: str)` and registered as an OpenAI function dict with `"name": "EndConversation"` (line 96).

**Gap:** One-shot mode calls the termination tool `reject_form` instead of `EndConversation`. To unify, `reject_form` should be renamed to `EndConversation` with the same `reason` field. The `fill_form` / `reject_form` pair can be kept as action types in the result schema, but the tool name presented to the LLM should be `EndConversation`.

### Marketplace

**File:** `packages/sage-benchmark/sage_benchmark/marketplace/environment/actions.py`

The termination tool is `EndNegotiation` (line 43):

```python
class EndNegotiation(Tool):
    reason: str = Field(min_length=1)
```

It inherits from the shared `Tool` base class and is included in the `MarketplaceAction` union (line 49-56) and `MARKETPLACE_TOOLS` list (line 58).

**Gap:** The tool is named `EndNegotiation` rather than `EndConversation`. To unify, rename the class to `EndConversation` (or alias it). The `reason` field already matches the target signature; the only difference is the `min_length=1` constraint, which is a reasonable addition and could be adopted across all benchmarks.
