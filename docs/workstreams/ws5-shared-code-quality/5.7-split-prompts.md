# 5.7 Split Prompts (Form-Filling)

## Definition & Architectural Decision

Form-filling's monolithic `prompts.py` should be split into separate files following the pattern established by calendar's `agents/assistant/prompts/` directory. Each prompt variant or functional group gets its own file for maintainability and easier diffing.

---

### Calendar (Reference Pattern)

**Location:** `calendar_scheduling/agents/assistant/prompts/` -- 6 separate files

Each file exports a single `SYSTEM_PROMPT` string constant:
- `default.py` (3 lines)
- `privacy_simple.py` (5 lines)
- `privacy_strong.py` (3 lines)
- `privacy_tom.py` (22 lines)
- `privacy_tom_dual.py` (31 lines)
- `privacy_ci.py` (56 lines)

The `__init__.py` provides `list_available_presets()` for auto-discovery. The CLI's `--assistant-system-prompt` flag uses this list for choices validation.

**Advantages of the split pattern:**
- Each preset is independently reviewable and diffable
- Adding a new preset requires only creating a new file (auto-discovered)
- No risk of accidentally breaking one prompt while editing another
- Clear file-level ownership for code review

### Marketplace

**Not applicable.** Marketplace has no prompts module. The only system prompt is a hardcoded string in `marketplace_base.py` (lines 47-51). Given marketplace's small prompt surface area, there is no need for a split-file pattern here.
