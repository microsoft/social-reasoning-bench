# 5.8 Audit: qwen_helpers

## Definition & Architectural Decision

Determine whether the `qwen_helpers` module is still actively used in the form-filling package or can be removed as dead code.

---

## Per-Package Analysis

### Calendar

**Not applicable.** No Qwen-related code exists in the calendar package. Grepping for `qwen` across `calendar_scheduling/` returns zero matches.

### Form-Filling

**Location:** `form_filling/qwen_helpers/` -- 4 source files + `__init__.py`

| File | Origin | Purpose |
|------|--------|---------|
| `schema.py` | Copied from `QwenLM/Qwen-Agent` | Message schema types for Qwen models |
| `fncall_prompt.py` | Copied from `QwenLM/Qwen-Agent` | Function-call prompt templates (`NousFnCallPrompt`) |
| `base_tools.py` | Copied from `QwenLM/Qwen-Agent` | Base tool definitions for Qwen agent framework |
| `utils.py` | Copied from `QwenLM/Qwen-Agent` | Utility functions |

All four files have header comments indicating they were copied from the Qwen-Agent repository.

**Active usage:** The `qwen_helpers` module is imported by exactly **one file**:

`form_filling/gui_prompt.py` (lines 5-7):
```python
from .qwen_helpers.base_tools import BaseTool
from .qwen_helpers.fncall_prompt import NousFnCallPrompt
from .qwen_helpers.schema import (
```

At line 410, `gui_prompt.py` uses the Qwen template:
```python
conversation = NousFnCallPrompt(template_name="qwen").preprocess_fncall_messages(
```

**No external imports:** Grepping for `qwen` outside the `form_filling/` package returns zero matches. The module is entirely internal to form-filling.

**Scope of dependency chain:**
- `gui_prompt.py` is imported by `gui.py` (line 27: `from sage_benchmark.form_filling.gui_prompt import build_gui_system_prompt`)
- `gui.py` is imported by `runner.py` (line 22: `from sage_benchmark.form_filling.gui import run_single_fara_task, start_http_server`)
- The GUI mode is selected via `--execution-mode gui` in the CLI

**Assessment:**
- `qwen_helpers` is **still used** but only for GUI mode (`--execution-mode gui`) through `gui_prompt.py`.
- The GUI mode targets `microsoft/Fara-7B` specifically (CLI default at line 210).
- The module contains vendored third-party code from the Qwen-Agent repository, which creates maintenance burden (no upstream updates, no version tracking).
- If GUI mode is deprecated or Fara-7B support is dropped, `qwen_helpers` can be removed entirely.
- The `NousFnCallPrompt` with `template_name="qwen"` is the only function actually called from the module.
