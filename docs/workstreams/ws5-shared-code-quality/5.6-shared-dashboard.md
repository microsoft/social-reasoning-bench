# 5.6 Shared Dashboard / Visualization

## Definition & Architectural Decision

A shared visualization module in `shared/dashboard/` enables cross-benchmark metric comparison. Currently each benchmark has its own plotting code with different data formats, chart types, and output styles. A unified dashboard would let researchers compare privacy-utility tradeoffs across calendar, form-filling, and marketplace experiments.

---

## Per-Package Analysis

### Calendar

**Location:** `calendar_scheduling/analysis/plot_pareto.py`

A matplotlib-based script that plots **Pareto fronts of scheduling success vs privacy preservation**:
- Scans output directories for `*.json` result files
- Extracts model name from filename using regex patterns (supports old and new naming conventions, lines 22-39)
- Plots task success rate vs privacy metrics
- Supports multiple runs overlaid on the same chart

**Data source:** `BenchmarkOutput` JSON files containing `BenchmarkSummary` with `task_success_rate`, `privacy_avg_leakage_rate`, `privacy_leakage_rate`, and fiduciary metrics.

### Form-Filling

**Multiple visualization files:**

#### `visualization_result.py` (2,719 lines)
A large HTML-generating visualization for **individual task results**:
- Detects mode (interactive, one-shot, GUI) from execution data (line 39-50)
- Renders mode-specific views: tool call timelines, conversation histories, final forms
- Supports `--all` flag for batch generation of per-task HTML files
- Can overlay evaluation data from separate eval results
- This is a **detailed inspection tool**, not a summary dashboard

#### `analysis/plot_results.py`
Matplotlib script for **aggregate results from summary.json files**:
- Finds `summary.json` files recursively in output directories (line 22-23)
- Extracts `agent_model`, `correctness`, `privacy`, and `validation` scores
- Plots bar charts comparing models

#### `analysis/compare_benchmarks.py`
Matplotlib script for **one-shot vs interactive comparison**:
- Loads summary files from two directories
- Extracts model, accuracy, and privacy scores
- Creates side-by-side comparison charts

### Marketplace

**Location:** `experiments/3-2-marketplace-metrics/plot_results.py`

A matplotlib script for **per-task marketplace metrics**:
- Loads results from `outputs/marketplace/3-2-marketplace-metrics/`
- Extracts variant data from summary JSON
- Plots deal rates, surplus shares, inference errors
- Hardcoded input/output paths (lines 18-19)

### Comparison Summary

| Feature | Calendar | Form-Filling | Marketplace |
|---------|----------|--------------|-------------|
| Summary plotting | `plot_pareto.py` | `plot_results.py` | `plot_results.py` |
| Cross-model comparison | Yes (Pareto front) | Yes (bar charts) | Yes (per-variant) |
| Individual result inspection | No | `visualization_result.py` (2,719 lines) | No |
| Benchmark comparison | No | `compare_benchmarks.py` | No |
| Output format | matplotlib PNG | matplotlib + HTML | matplotlib PNG |
| Data format | `BenchmarkOutput` JSON | `summary.json` | Custom `results.json` |
| Location | In-package `analysis/` | In-package + `visualization_result.py` | In `experiments/` directory |

**Key challenges for unification:**
- Data formats differ significantly: calendar uses `BenchmarkOutput` with `BenchmarkSummary`; form-filling uses flat `summary.json`; marketplace uses custom per-variant results.
- Metric names differ: calendar has `task_success_rate` and `privacy_avg_leakage_rate`; form-filling has `aggregate_correctness_score` and `aggregate_privacy_score`; marketplace has `deal_rate` and `inference_error`.
- The `visualization_result.py` file at 2,719 lines is an outlier -- it is an HTML report generator, not a metrics dashboard.
- Marketplace plotting lives outside the package in `experiments/`, suggesting it was added ad-hoc.
